# ZC300 系列 Modbus/RTU 通信配置注意事项

在配置 ZC300 系列控制器进行 Modbus/RTU 通信前，请务必关注以下关键信息：

1. **物理连接**：采用 RS-485 两线制半双工接线方式，构建 RTU 通讯网络时，**在总线最末端的从站设备两端需并联一个 120Ω 的终端电阻**。
2. **通讯参数**：
   - **主站**：波特率必须和从站一致（数据位:8，停止位:1，无校验）。
   - **从站(ZC300)**：**波特率固定为 115200**。默认 32 位数据采用高位在前（大端序）。从站地址可通过 LCD 面板配置，范围 1~255。
3. **功能码**：支持 `0x03`（读保持寄存器）、`0x04`（读输入寄存器）、`0x06`（写单个寄存器）、`0x10`（写多个寄存器）。
4. **寄存器地址映射转换（⚠️ 极易出错）**：手册中提到的寄存器编号（如30050）为物理地址号，实际 Modbus 通讯报文中的 **寄存器地址 = 寄存器编号 - 1**。例如，操作码寄存器 30050 转成报文地址为 `30050-1 = 30049 = 0x7561`。
5. **指令与通信机制**：采用主问从答的查询-反馈机制。发送控制指令时需将控制字（操作码）写入 `30050` 寄存器，操作码及配套参数写入 `30051~30053` 寄存器。操作码存入后不保存，掉电丢失。
6. **异常排查**：如反馈异常响应码，请注意：
   - `0x06`：指令报警（若当前轴正在运动，禁止发送新的运动指令）。
   - `0x07`：限位起效。
   - `0x08`：急停起效。
   - `0x09`：未使能。
   - `0x0A`：操作码错误。
7. **参数保存**：修改参数（如速度、距离、细分等）后，若要永久保存，必须在**三轴均处于静止状态**下执行 `0x006D` 保存参数操作码。若轴处于运动状态，直接保存会返回存储失败。

# 附录3：ZC300系列 Modbus/RTU 寄存器表

> **寄存器地址范围说明**：`30001 ~ 30049` 为**输入寄存器**（只读，对应功能码 `0x04`）；`30050 ~ 30140` 为**保持寄存器**（可读写，对应功能码 `0x03`/`0x06`/`0x10`）。
> 单位 `pp` 代表 Pulses（脉冲）。以下所涉及浮点数/长整型，在报文中通常占用两个连续的寄存器。

| 寄存器地址 | 读写属性 | 数据类型 | 描述 | 详细备注与取值范围 |
| :--- | :--- | :--- | :--- | :--- |
| **30001~30007** | 只读 | 字符串 | 设备型号 | 用于读取产品型号名称。 |
| **30008~30009** | 只读 | LONG | 设备编号 | 十进制显示。例如读取值为 `0x013463A9`，即十进制 `20210601`。 |
| **30010** | 只读 | SHORT | 软件版本 | 数值保留一位小数点显示，如 `0x0001` 代表 `v1.0`。 |
| **30011** | 只读 | SHORT | IO口输入状态 | 位映射：Bit0=IN1, Bit1=IN2, Bit2=IN3。1 表示高电平，0 表示低电平。 |
| **30012~30014** | 只读 | SHORT | X/Y/Z轴运动状态 | `0x01`：轴正在运动中；`0x00`：轴处于停止状态。 |
